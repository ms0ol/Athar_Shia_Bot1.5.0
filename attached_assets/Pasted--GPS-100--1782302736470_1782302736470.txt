مشاركة الموقع الجغرافي (GPS) مع بوت التليجرام هي الخطوة المثالية لتوفير مواقيت صلاة دقيقة 100% لكل مستخدم بحسب مكانه الفعلي (سواء كان في مركز الكوت، قضاء بدرة، أو أي مكان آخر).
إليك أفضل الحلول والأفكار البرمجية لتطبيق هذه الميزة في بوت **أثَر الشيعة** باستخدام مكتبة **Aiogram**:
## 1. الحل البرمجي الأفضل: زر إرسال الموقع بنقرة واحدة (Keyboard Button)
أسهل طريقة وأكثرها سلاسة للمستخدم هي وضع زر مخصص في قائمة الإعدادات يطلب منه مشاركة موقعه بنقرة واحدة. التليجرام سيتكفل بإرسال إحداثيات الـ GPS الدقيقة فوراً.
### كود إنشاء الزر (Keyboards):
عندما يدخل المستخدم لإعدادات الموقع، تعرض له لوحة مفاتيح تحتوي على زر مشاركة الموقع:
```python
from aiogram.types import ReplyKeyboardMarkup, KeyboardButton

def get_location_keyboard():
    keyboard = ReplyKeyboardMarkup(resize_keyboard=True, one_time_keyboard=True)
    # هذا الزر يطلب من التليجرام فتح الـ GPS وإرسال الموقع مباشرة
    btn_location = KeyboardButton(text="📍 مشاركة موقعي الحالي (GPS)", request_location=True)
    btn_back = KeyboardButton(text="🔙 إلغاء")
    
    keyboard.add(btn_location)
    keyboard.add(btn_back)
    return keyboard

```
### كود معالجة الموقع المستلم (Handler):
في ملف handlers.py، تقوم بإنشاء معالج يستقبل رسائل الموقع (content_types=['location'])، ومن ثم تأخذ الإحداثيات وتخزنها في قاعدة البيانات للمستخدم:
```python
from aiogram import types
import database as db # فرضاً أن لديك دالة لتحديث الموقع

async def handle_user_location(message: types.Message):
    if message.location:
        lat = message.location.latitude
        lng = message.location.longitude
        user_id = message.from_user.id
        
        # 1. حفظ الإحداثيات الدقيقة في قاعدة البيانات للمستحدم
        db.update_user_location(user_id, lat, lng, city_name="موقعي المخصص")
        
        # 2. إزالة لوحة المفاتيح العادية وإرجاع قائمة inline
        await message.answer(
            f"✅ تم تحديث موقعك بنجاح!\n"
            f"📍 الإحداثيات الحالية:\n"
            f"العرض (Lat): {lat:.4f}\n"
            f"الطول (Lng): {lng:.4f}\n\n"
            f"ستعتمد مواقيت الصلاة الآن على موقعك الفعلي بدقة تامة.",
            reply_markup=types.ReplyKeyboardRemove() # لإخفاء زر الـ GPS بعد الاستخدام
        )

```
## 2. أفكار وحلول ذكية لتجربة مستخدم (UI/UX) احترافية
إذا كنت تريد جعل البوت ذكياً وتفاعلياً، إليك هذه الأفكار لتطبيقها مع ميزة الـ GPS:
### أ. فكرة "المنطقة الزمنية التلقائية"
بدل أن يختار المستخدم المنطقة الزمنية يدوياً (Asia/Baghdad)، يمكنك استخدام مكتبات بايثون مثل timezonefinder. بمجرد أن يرسل المستخدم موقعه الـ GPS، يمرر البوت الإحداثيات للمكتبة لتجلب له المنطقة الزمنية الصحيحة وتثبيتها تلقائياً في الإعدادات.
### ب. دعم الطريقتين (GPS + القائمة اليدوية)
بعض المستخدمين قد يرفضون مشاركة الـ GPS لأسباب تتعلق بالخصوصية أو لاستخدامهم تليجرام من الكمبيوتر. اجعل خيار الـ GPS هو **الخيار الموصى به أولاً**، وضع أسفله خياراً آخراً: "أو اختر المحافظة يدويًا" (والتي تعرض قائمة المدن التقليدية).
### ج. ميزة الجيودرينج العكسي (Reverse Geocoding)
عندما يرسل المستخدم إحداثيات مثل 32.5000 و 45.8167 عبر الـ GPS، يمكنك استخدام مكتبة مجانية مثل geopy لمعرفة اسم المنطقة وعرضها للمستخدم، ليظهر له البوت رسالة تأكيدية ذكية:
> "تم تحديد موقعك في: **واسط - قضاء بدرة**. هل تريد اعتماد هذه المنطقة؟"
