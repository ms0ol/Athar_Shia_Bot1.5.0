### 🛠️ الحل الجذري والملف البرمجي الذي يحتاج تعديل
لجعل البوت يرسل ملف الـ PDF الخاص بالدعاء مباشرة للمستخدم من قناة التليجرام باستخدام الـ file_id الموجود في ملفك الجديد، يجب عليك **تعديل دالة callback_random_dua في ملف handlers.py** لتصبح ذكية وتعرف كيف ترسل الملفات:
قم بفتح ملف **handlers.py** وابحث عن دالة callback_random_dua واستبدلها بالكامل بهذا الكود:
```python
async def callback_random_dua(call: CallbackQuery):
    """Handle random dua supporting both Text and PDF files from JSON."""
    user_id = call.from_user.id
    item = get_random_item("daily_dua", user_id)
    
    if not item:
        await call.answer("🤲 عذراً، لم يتم العثور على أدعية متوفرة حالياً.", show_alert=True)
        return

    # تسجيل أن المستخدم استلم هذا الدعاء حتى لا يتكرر له بناءً على نظام البوت
    db.mark_content_sent(user_id, "daily_dua", item["id"])

    # 1. التحقق إذا كان الدعاء عبارة عن ملف PDF ويحتوي على file_id (مثل ملفك الجديد)
    if item.get("is_pdf") and item.get("file_id"):
        # نقوم بإرسال الملف كرسالة مستقلة للمستخدم باستخدام معرف الملف
        await call.message.answer_document(
            document=item["file_id"],
            caption=f"📿 <b>{item.get('title', 'دعاء يومي')}</b>\n\n نسألكم الدعاء.",
            parse_mode="HTML"
        )
        # إشعار منبثق سريع بنجاح الإرسال
        await call.answer("تم إرسال ملف الدعاء بنجاح ✅")
    
    # 2. إذا كان الدعاء نصياً قديمًا يحتوي على نص
    elif item.get("text"):
        text = format_dua(item)
        await call.message.edit_text(text, parse_mode="HTML", reply_markup=back_button("menu:library"))
        await call.answer()
        
    else:
        await call.answer("⚠️ خطأ في صيغة بيانات الدعاء داخل الخادم.", show_alert=True)

```
### 💡 لماذا هذا التعديل سيحل المشكلة؟
 1. **يفحص البيانات أولاً:** الكود الجديد سيتحقق عبر شرط if item.get("is_pdf") إذا كان العنصر ملفاً.
 2. **يستخدم الدالة الصحيحة:** بدلاً من محاولة تعديل النص بـ edit_text (والتي تسبب الاختفاء أو الخطأ)، سيستخدم دالة answer_document المخصصة لإرسال المستندات والملفات المرفوعة على سيرفرات التليجرام مباشرة عبر الـ file_id.
بمجرد استبدال هذه الدالة في ملف الـ handlers.py وإعادة تشغيل البوت، سيبدأ بإرسال ملفات الأدعية الـ PDF المذكورة في ملف الـ JSON فوراً وبدون أي مشاكل!
