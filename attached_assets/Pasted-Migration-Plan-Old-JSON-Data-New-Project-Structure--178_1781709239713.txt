Migration Plan — Old JSON Data → New Project Structure

أريد تنفيذ عملية ترحيل كاملة لبيانات المشروع القديم إلى هيكل المشروع الجديد.

مهم جداً:

* لا تقم بإنشاء بيانات وهمية.
* لا تكتب أي محتوى تجريبي.
* اعمل فقط على تحويل ودمج البيانات الموجودة فعلياً.
* حافظ على جميع النصوص الأصلية كما هي.
* حافظ على جميع المعرفات IDs إن كانت موجودة.
* لا تحذف أي عنصر إلا إذا كان مكرراً حرفياً.

⸻

الهدف

لدينا:

1- ملفات JSON قديمة تحتوي بيانات حقيقية.

2- مشروع جديد مبني على هيكل أحدث وأفضل.

المطلوب:

تحويل البيانات القديمة إلى الهيكل الجديد مع المحافظة على كل المحتوى.

pdf_library.json

سيكون المستودع المركزي لجميع ملفات PDF.

الصيغة:

{
  "metadata": {},
  "items": []
}

كل عنصر:

{
  "id": "",
  "category": "",
  "title": "",
  "telegram_file_id": "",
  "description": ""
}

الفئات المسموحة:

* dua
* ziyarah
* munajat
* book
* calendar

لا تضف أي عنصر فارغ.

إذا لا توجد بيانات حالياً فاترك items فارغة.

Validation

بعد الترحيل:

1. تحقق من صحة جميع ملفات JSON.
2. تأكد أن:

json.load(...)

يعمل على جميع الملفات بدون استثناء.

3. احسب:

* عدد الأحاديث
* عدد الحكم
* عدد الأدعية
* عدد المناسبات
* عدد التعقيبات

واعرض تقريراً نهائياً.

⸻

المطلوب النهائي

أريد في النهاية:

1. جميع البيانات القديمة منقولة.
2. جميع الملفات بصيغة الهيكل الجديد.
3. عدم فقدان أي محتوى.
4. عدم إنشاء بيانات وهمية.
5. تقرير Migration Report يوضح:
    * الملفات التي تم دمجها.
    * الملفات التي تم إنشاؤها.
    * عدد العناصر في كل ملف.
    * أي أخطاء أو عناصر تم تجاهلها.
